Nuevos Paradigmas de Almacenamiento

MongoDB y NoSQL

No todos los datos del censo de personas encajan en tablas perfectas. ¿Qué pasa si una persona tiene 3 números de teléfono, 5 hobbies y otra persona no tiene ninguno? Aquí entran las bases de datos orientadas a documentos (JSON): MongoDB.

4.1

Flexibilidad JSON y MongoDB Shell (mongosh)

En MongoDB, en lugar de Tablas y Filas, tenemos Colecciones y Documentos. Cada documento es un objeto JSON independiente. Esta flexibilidad permite evolucionar rápidamente el esquema de nuestros datos (Ideal para Big Data semi-estructurado).

Instalación de MongoDB Shell (mongosh)

Para comunicarnos con cualquier base de datos de MongoDB desde nuestra terminal, necesitamos su intérprete de comandos.

  1. Descarga mongosh desde la página oficial de descargas de MongoDB (apto para Windows, Mac y Linux).
  2. Instala o descomprime el archivo y asegúrate de agregar la ruta a las variables de entorno (PATH) de tu sistema.
  3. Abre tu terminal y escribe mongosh para verificar la instalación. Si ves la versión impresa, ¡estás listo!

Comandos Básicos de Comunicación

En la consola interactiva (mongosh), usamos sintaxis de JavaScript para crear e interactuar con nuestros datasets.

>_ mongosh 
// 1. Usar o crear la base de datos
use censo_nacional_db;

// 2. Insertar un dataset de personas (Documentos JSON)
db.personas.insertMany([
  { 
    id_persona: "A100", 
    nombre: "Laura Sanchez", 
    edad: 25, 
    ciudad: "Monterrey",
    habilidades_tech: ["Python", "GCP", "NoSQL"] // ¡Un arreglo dentro del registro!
  },
  { 
    id_persona: "A101", 
    nombre: "Roberto Diaz", 
    edad: 42, 
    ciudad: "CDMX",
    vehiculo: { tipo: "Auto", año: 2020 } // ¡Un objeto anidado, y sin habilidades_tech!
  }
]);

// 3. Consultar todos los registros de Monterrey
db.personas.find({ ciudad: "Monterrey" });

// 4. Actualizar agregando un nuevo campo dinámicamente
db.personas.updateOne(
  { nombre: "Laura Sanchez" }, 
  { $set: { estado_civil: "Soltera" } }
);
4.2

Atlas y Carga de Datasets (CSV)

Paso a paso: Creación de Clúster en MongoDB Atlas

MongoDB Atlas es la versión 100% administrada en la nube (DBaaS). Evitamos configurar servidores locales y ganamos accesibilidad global.

  1. Ve a MongoDB Atlas y regístrate para obtener una cuenta gratuita.
  2. Haz clic en Build a Database. Selecciona el plan M0 (Free).
  3. Elige tu proveedor en la nube. Como estamos trabajando con Google, selecciona Google Cloud y una región cercana (ej. us-central1). Haz clic en Create.
  4. Seguridad - Crear Usuario: En la pestaña Database Access, crea un usuario. Llamémoslo admin_censo con una contraseña segura.
  5. Seguridad - Acceso de Red: En Network Access, haz clic en Add IP Address y selecciona Allow Access from Anywhere (0.0.0.0/0). (Nota: Esto es para nuestra práctica, en producción solo agregarías IPs confiables).
  6. Ve a Database y haz clic en Connect. Selecciona Connect with the MongoDB Shell para obtener tu Connection String.
mongodb+srv://admin_censo:TU_PASSWORD@cluster0.abcde.mongodb.net/

Paso a paso: Subir nuestro CSV a Atlas (mongoimport)

Tenemos nuestro archivo datos_personas.csv de la Unidad 2. ¿Cómo lo subimos masivamente a Atlas? Usaremos la herramienta de línea de comandos mongoimport (incluida en las MongoDB Database Tools).

  1. Abre tu terminal del sistema operativo (CMD o Bash) en la carpeta donde tienes tu archivo datos_personas.csv.
  2. Asegúrate de tener instaladas las MongoDB Database Tools (se descargan desde la página oficial y se agregan al PATH).
  3. Ejecuta el siguiente comando, reemplazando la URI con tu cadena de conexión de Atlas:
mongoimport --uri "mongodb+srv://admin_censo:TU_PASSWORD@cluster0.abcde.mongodb.net/" \
  --db censo_nacional_db \
  --collection personas \
  --type csv \
  --headerline \
  --file datos_personas.csv

El parámetro --headerline le indica a Mongo que use la primera fila del CSV como los nombres de los campos (id_persona, nombre, etc.) para armar los JSON automáticamente.

Conexión Externa a Looker Studio

Ya vimos cómo conectar BigQuery. Ahora conectaremos nuestro clúster de Atlas a Looker Studio. Como los datos NoSQL vienen en formato JSON (anidados y sin esquema rígido), Looker requiere un conector especial que "traduzca" el formato a columnas planas (SQL) para poder hacer gráficos. Usaremos el conector oficial MongoDB Atlas SQL.

Paso a paso: Enlazando las plataformas

  1. Preparar Federated Queries en Atlas: Antes de conectar Looker, Atlas necesita saber cómo traducir tu JSON a SQL. En tu panel de MongoDB Atlas, ve a Data Federation en el menú lateral y haz clic en Create Federated Database. Vincula tu base censo_nacional_db.
  2. Ir a Looker Studio: Abre Looker Studio, haz clic en "Informe vacío" o abre tu dashboard de la Unidad 3 y selecciona Añadir Datos.
  3. Buscar el Conector: En la barra de búsqueda de conectores, escribe MongoDB Atlas SQL (Desarrollado por MongoDB Inc.). Haz clic en él y autoriza el acceso a tu cuenta de Google.
  4. Llenar Credenciales: El conector te pedirá 3 datos vitales:
    • Federated Database Name: El nombre de la federación que creaste en el paso 1 (ej. Federation-Censo).
    • Username: admin_censo
    • Password: Tu contraseña.
  5. ¡Graficar!: Haz clic en Añadir. Looker Studio ahora mapeará los campos de tus documentos MongoDB (como edad, ciudad) como si fueran columnas tradicionales. Ya puedes arrastrar gráficos de barras y pasteles mezclando datos transaccionales NoSQL con tu analítica en la nube.

Retos Prácticos

Reto 1: Consultas MQL

Dificultad: Media

Basado en la colección personas, escribe el comando exacto en mongosh para buscar a todas las personas cuya edad sea mayor a 30 años.

Pista: En MQL (Mongo Query Language), los operadores van con un signo de dólar. Usa $gt (greater than). 

db.personas.find(
  { edad: { $gt: 30 } }
);

Explicación: En lugar del WHERE edad > 30 de SQL, MongoDB usa JSON. Declaramos el campo (edad) y como valor pasamos otro sub-objeto indicando la comparación ($gt).

Reto 2: Ingesta de CSV

Dificultad: Fácil

Estás subiendo tu archivo nuevo_censo.csv a Atlas. Faltan dos parámetros clave en el comando para indicar la base de datos y decirle al sistema que ignore la primera fila asumiendo que son las cabeceras.

mongoimport --uri "mongodb+srv://..." \
  --____ censo_nacional_db \
  --collection personas \
  --type csv \
  --____ \
  --file nuevo_censo.csv
  • --db (Para especificar la Database)
  • --headerline (Para tomar la primera fila como nombres de campo)

2.1

Bodegas de Datos (Datawarehouse)

Un Datawarehouse (DW) es un sistema centralizado que consolida datos de múltiples fuentes heterogéneas. Su propósito único es permitir consultas y análisis rápidos para la toma de decisiones empresariales.

Implementación en GCP con Python

En Google Cloud, nuestro Datawarehouse es BigQuery. Así es como cargaríamos datos crudos hacia el DW usando el SDK de Python:

python cargar_personas_dw.py 
from google.cloud import bigquery

# 1. Conectar al Datawarehouse (BigQuery)
client = bigquery.Client(project='mi-proyecto-123')

# 2. Definir la tabla destino en el DW
table_id = "mi-proyecto-123.dataset_personas.directorio_nacional"

# 3. Configurar el trabajo (Ej. cargar desde un archivo local o Cloud Storage)
job_config = bigquery.LoadJobConfig(
    source_format=bigquery.SourceFormat.CSV,
    skip_leading_rows=1,
    autodetect=True, # Infiere las columnas automáticamente
)

# 4. Ejecutar la carga (Fase 'Load' del ETL)
with open("datos_personas.csv", "rb") as source_file:
    job = client.load_table_from_file(source_file, table_id, job_config=job_config)

job.result()  # Esperar a que termine
print(f"¡Carga exitosa! Hay {client.get_table(table_id).num_rows} personas en el Datawarehouse.")

Procesamiento de Transacciones en Línea (OLTP)

OLTP (Online Transaction Processing) es el sistema del "día a día". Controla las operaciones cotidianas en tiempo real: una compra en Amazon, un retiro de cajero automático, un registro médico.

Su objetivo es la Velocidad de Escritura y la Integridad (ACID).

A
Atomicidad
Todo o nada
C
Consistencia
Reglas válidas
I
Aislamiento
Independientes
D
Durabilidad
No se borra

Ejemplo de consulta OLTP (Cloud SQL - MySQL)

Busca o inserta un registro específico muy rápido por su ID. Ideal para registrar una nueva persona en el sistema.

-- El sistema registra a una nueva persona
START TRANSACTION;

INSERT INTO personas (id_persona, nombre, edad, ciudad, fecha_registro) 
VALUES (8492, 'Ana Garcia', 28, 'CDMX', NOW());

UPDATE estadisticas_ciudad 
SET total_habitantes = total_habitantes + 1 
WHERE ciudad = 'CDMX';

COMMIT; -- Cumple propiedades ACID

Paso a paso en BigQuery: Consultando OLTP (Consultas Federadas)

Aunque BigQuery no es para OLTP, a menudo necesitamos consultar la base de datos operativa (ej. Cloud SQL) en tiempo real sin tener que duplicar los datos. Esto se logra mediante Consultas Federadas.

  1. Ve a la consola de BigQuery en Google Cloud.
  2. En el panel Explorador (lado izquierdo), haz clic en el botón + Agregar datos y selecciona Conexiones externas.
  3. Elige el origen (ej. Cloud SQL para MySQL) y configura las credenciales de tu base de datos OLTP.
  4. Abre una nueva pestaña del editor de consultas en BigQuery.
  5. Escribe una consulta usando la función EXTERNAL_QUERY() para leer la base transaccional en vivo y haz clic en EJECUTAR.
-- Ejecutar esto en BigQuery para leer tu BD transaccional (OLTP)
SELECT * FROM EXTERNAL_QUERY(
    "mi-proyecto.us-central1.mi_conexion_sql", 
    "SELECT id_persona, nombre, ciudad FROM personas WHERE fecha_registro = CURDATE();"
);
2.3

Procesamiento y Análisis en Línea (OLAP)

OLAP (Online Analytical Processing) es el sistema del "gerente". Analiza volúmenes masivos de datos históricos para responder preguntas complejas de negocio. Funciona sobre el Datawarehouse.

Su objetivo es la Velocidad de Lectura Masiva (Agrupaciones/Filtros).

El secreto de OLAP: Almacenamiento Columnar

A diferencia de OLTP que guarda por filas, OLAP guarda por columnas. Esto permite promediar edades de millones de personas en segundos sin leer sus nombres o correos.

Ejemplo de consulta OLAP (BigQuery)

Lee millones de registros de personas, los agrupa por ciudad y calcula métricas demográficas (Analítica pesada).

-- El gerente quiere ver la distribución demográfica
SELECT 
    ciudad,
    COUNT(id_persona) AS total_personas,
    AVG(edad) AS edad_promedio
FROM `dataset_personas.directorio_nacional`
GROUP BY ciudad
ORDER BY total_personas DESC;

Paso a paso en BigQuery: Ejecutando Análisis Masivo (OLAP)

Aquí es donde BigQuery brilla. Vamos a ejecutar una consulta sobre millones de registros históricos que ya están en nuestro Datawarehouse.

  1. Ingresa a tu BigQuery Workspace en Google Cloud.
  2. En el Explorador, despliega tu Proyecto > tu Dataset (ej. dataset_personas) y haz clic en tu tabla (ej. directorio_nacional).
  3. Revisa la pestaña Esquema (Schema) para ver los nombres y tipos de datos de cada columna.
  4. Haz clic en el botón azul + REDACTAR CONSULTA nueva (Compose new query).
  5. Pega tu consulta SQL (como la del ejemplo de arriba) en el área de texto.
  6. ¡Control de Costos! Observa la esquina superior derecha del editor. Verás una palomita verde indicando: "Esta consulta procesará X GB cuando se ejecute".
  7. Haz clic en EJECUTAR (Run) y explora los resultados en el panel inferior.

Paso a paso en BigQuery: Exportación y Visualización

El ciclo OLAP termina cuando entregamos valor. Una vez que la interfaz gráfica te devuelve los resultados, el siguiente paso es dárselos a los tomadores de decisiones.

  1. Después de ejecutar tu consulta, ubícate en el panel inferior de Resultados de la consulta.
  2. Busca y haz clic en el botón GUARDAR RESULTADOS.
  3. Selecciona CSV (archivo local) para descargar, o Google Sheets para compartir la tabla en la nube.
  4. Para ir al siguiente nivel: Haz clic en el botón EXPLORAR DATOS (junto a Guardar Resultados).
  5. Selecciona Explorar con Looker Studio. GCP abrirá automáticamente una nueva pestaña conectada a tus datos lista para arrastrar gráficos y crear un dashboard.

Reto 1: Consulta Analítica (SQL)

Dificultad: Media

Tienes la tabla directorio_nacional en BigQuery con los campos: id_persona, nombre, profesion y estado_civil.

Tu tarea: Escribe una consulta SQL para contar cuántas personas hay agrupadas por profesion, pero solo para aquellas cuyo estado_civil sea 'Casado'. Ordena de mayor a menor cantidad. 

SELECT 
    profesion, 
    COUNT(*) AS total_personas
FROM `tu_proyecto.dataset_personas.directorio_nacional`
WHERE estado_civil = 'Casado'
GROUP BY profesion
ORDER BY total_personas DESC;

Explicación: Filtramos primero (WHERE estado_civil = 'Casado'), luego agrupamos por profesión (GROUP BY) y usamos la función de agregación COUNT() para contar las personas en cada grupo.

Paso a paso en BigQuery: Cómo probar retos SQL hoy mismo

Si aún no tienes datos en tu proyecto, Google ofrece petabytes de datos reales gratuitos (Public Datasets) para practicar estos retos y consultas OLAP.

  1. En la consola de BigQuery, ve al panel izquierdo (Explorador) y haz clic en + AGREGAR.
  2. Selecciona la opción Conjuntos de datos públicos (Public Datasets).
  3. En el buscador, escribe Census Bureau (Censo) y presiona enter.
  4. Añádelo a tu proyecto (Aparecerá en el árbol del explorador con el nombre bigquery-public-data).
  5. Abre una consulta nueva y adapta tu SQL para leer esta tabla demográfica real. Ejemplo:
    SELECT * FROM `bigquery-public-data.census_bureau_usa.population_by_zip_2010` LIMIT 10;

2.1

Bodegas de Datos (Datawarehouse)

Un Datawarehouse (DW) es un sistema centralizado que consolida datos de múltiples fuentes heterogéneas. Su propósito único es permitir consultas y análisis rápidos para la toma de decisiones empresariales.

Implementación en GCP con Python

En Google Cloud, nuestro Datawarehouse es BigQuery. Así es como cargaríamos datos crudos hacia el DW usando el SDK de Python:

python cargar_dw.py 
from google.cloud import bigquery

# 1. Conectar al Datawarehouse (BigQuery)
client = bigquery.Client(project='mi-proyecto-123')

# 2. Definir la tabla destino en el DW
table_id = "mi-proyecto-123.ventas_dataset.historico_ventas"

# 3. Configurar el trabajo (Ej. cargar desde un archivo local o Cloud Storage)
job_config = bigquery.LoadJobConfig(
    source_format=bigquery.SourceFormat.CSV,
    skip_leading_rows=1,
    autodetect=True, # Infiere las columnas automáticamente
)

# 4. Ejecutar la carga (Fase 'Load' del ETL)
with open("ventas_2025.csv", "rb") as source_file:
    job = client.load_table_from_file(source_file, table_id, job_config=job_config)

job.result()  # Esperar a que termine
print(f"¡Carga exitosa! Hay {client.get_table(table_id).num_rows} filas en el Datawarehouse.")

Procesamiento de Transacciones en Línea (OLTP)

OLTP (Online Transaction Processing) es el sistema del "día a día". Controla las operaciones cotidianas en tiempo real: una compra en Amazon, un retiro de cajero automático, un registro médico.

Su objetivo es la Velocidad de Escritura y la Integridad (ACID).

A
Atomicidad
Todo o nada
C
Consistencia
Reglas válidas
I
Aislamiento
Independientes
D
Durabilidad
No se borra

Ejemplo de consulta OLTP (Cloud SQL - MySQL)

Busca un registro específico muy rápido por su ID (Lectura de 1 fila) o inserta una venta.

-- El usuario da clic en "Comprar"
START TRANSACTION;

INSERT INTO ordenes (id_cliente, total, fecha) 
VALUES (4589, 1200.50, NOW());

UPDATE inventario 
SET stock = stock - 1 
WHERE id_producto = 'LAPTOP-01';

COMMIT; -- Cumple propiedades ACID

2.3

Procesamiento y Análisis en Línea (OLAP)

OLAP (Online Analytical Processing) es el sistema del "gerente". Analiza volúmenes masivos de datos históricos para responder preguntas complejas de negocio. Funciona sobre el Datawarehouse.

Su objetivo es la Velocidad de Lectura Masiva (Agrupaciones/Filtros).

El secreto de OLAP: Almacenamiento Columnar

A diferencia de OLTP que guarda por filas, OLAP guarda por columnas. Esto permite sumar millones de precios en segundos sin leer nombres o direcciones.

Ejemplo de consulta OLAP (BigQuery)

Lee millones de registros, los agrupa y calcula promedios (Analítica pesada).

-- El gerente quiere ver tendencias anuales
SELECT 
    EXTRACT(MONTH FROM fecha) AS mes,
    categoria,
    SUM(total) AS ingresos_totales,
    AVG(total) AS ticket_promedio
FROM `historico_ventas`
WHERE EXTRACT(YEAR FROM fecha) = 2025
GROUP BY mes, categoria
ORDER BY ingresos_totales DESC;

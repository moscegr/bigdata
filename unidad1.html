1.1

Definición y Características

Big Data no es solo "muchos datos". Se refiere a conjuntos de datos cuyo volumen, velocidad o variedad es tan grande que dificulta su captura, gestión, procesamiento o análisis mediante tecnologías convencionales (como una base de datos SQL tradicional o Excel).

¿Sabías que? Google fue pionero en esto. Para indexar todo internet, inventaron tecnologías como Google File System y MapReduce, que luego inspiraron a Hadoop.

1.2

Tipos de Datos

En Big Data, raramente trabajamos con tablas perfectas. Clasificamos los datos en tres categorías:

Estructurados

Formato fijo (Filas/Columnas)

Ejemplo: Tu boleta de calificaciones en el sistema escolar.

Semi-Estructurados

Tienen etiquetas pero no esquema rígido

Ejemplo: Un tweet (tiene texto, autor, fecha en formato JSON).

No Estructurados

Binarios o texto libre. El 80% de los datos.

Ejemplo: Una nota de voz de WhatsApp o un video de YouTube.
ejemplo_datos.json (Semi-estructurado) JSON 
{
  "sensor_id": "IoT-291",
  "timestamp": "2026-02-15T14:30:00Z",
  "lecturas": {
    "temperatura": 24.5,
    "vibracion": [0.1, 0.4, 0.0]
  },
  "estado": "ACTIVO"
}

Nota GCP: Este tipo de dato JSON es nativo para herramientas como Google BigQuery, donde podemos consultarlo directamente.

1.4

Modelos y Tareas

En Big Data, definimos las tareas basándonos en la madurez analítica. No es lo mismo saber qué pasó ayer, que saber qué pasará mañana.

Pasado 1. Análisis Descriptivo (¿Qué pasó?)

Es la base. Resume datos históricos para entender el estado actual.
  • Ejemplo: Reporte de ventas mensuales o dashboard de visitas web.
  • Tarea: Agregación, Filtrado, Visualización.
  • Looker Studio

Futuro 2. Análisis Predictivo (¿Qué pasará?)

Usa datos históricos y algoritmos para predecir eventos futuros con cierta probabilidad.
  • Ejemplo: Predicción de demanda de inventario para Navidad.
  • Tarea: Regresión, Series de Tiempo.
  • BigQuery ML

Acción 3. Análisis Prescriptivo (¿Qué debemos hacer?)

No solo predice, sino que sugiere la mejor acción a tomar. Es el nivel más avanzado.
  • Ejemplo: Waze no solo dice "hay tráfico" (Descriptivo) o "llegarás tarde" (Predictivo), sino que te cambia la ruta automáticamente (Prescriptivo).
  • Tarea: Optimización, Simulación.
  • Vertex AI Optimization
1.5

Métodos y Técnicas

Aquí entramos en el "cómo". Estas son las técnicas matemáticas y algorítmicas (Machine Learning) que aplicamos a los datos.

Clasificación

Asigna una etiqueta a un dato nuevo basándose en el aprendizaje previo.

input: Email
output: "Spam" o "No Spam"

Regresión

Predice un valor numérico continuo.

input: Metros cuadrados, Zona
output: Precio Casa ($)

Clustering (Agrupamiento)

Agrupa datos similares sin etiquetas previas (No Supervisado).

input: Base de Clientes
output: Grupo A (Jóvenes), Grupo B (VIP)

Reglas de Asociación

Descubre relaciones ocultas entre variables ("Si compra A, compra B").

input: Tickets de Walmart
output: {Pañales} -> {Cerveza}

1.9 - 1.10

Gobernanza y Retos

Gobernanza de Datos

Es la gestión de la disponibilidad, integridad y seguridad. En GCP usamos IAM para permisos y Data Plex para gobernanza.

  • Calidad: Datos limpios y fiables.
  • Seguridad: ¿Quién accede a qué?
  • Privacidad: Cumplimiento legal (GDPR).

Problemática

No todo es perfecto. Implementar Big Data trae problemas reales.

  • Escasez de talento (Faltan Ingenieros de Datos).
  • Costos si no se optimiza la nube.
  • Datos sucios ("Garbage in, Garbage out").

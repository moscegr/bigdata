1.1

Definición y Características

Big Data no es solo "muchos datos". Se refiere a conjuntos de datos cuyo volumen, velocidad o variedad es tan grande que dificulta su captura, gestión, procesamiento o análisis mediante tecnologías convencionales (como una base de datos SQL tradicional o Excel).

Concepto clave: El umbral de Big Data es relativo y cambia con el tiempo. Lo que hoy es Big Data, mañana puede ser procesado por tu smartphone.

1.2

Tipos de Datos

En Big Data, raramente trabajamos con tablas perfectas. Clasificamos los datos en tres categorías:

Estructurados

Formato fijo (Filas/Columnas)

Semi-Estructurados

Tienen etiquetas pero no esquema rígido

No Estructurados

Binarios o texto libre. El 80% de los datos.

ejemplo_datos.json (Semi-estructurado) JSON 
{
  "sensor_id": "IoT-291",
  "timestamp": "2026-02-15T14:30:00Z",
  "lecturas": {
    "temperatura": 24.5,
    "vibracion": [0.1, 0.4, 0.0]
  },
  "estado": "ACTIVO"
}

Nota como los datos pueden anidar arrays y objetos, algo difícil en una tabla plana.

1.3

Áreas y Aplicaciones

Marketing y Ventas

Segmentación de clientes, predicción de abandono (churn), recomendación de productos (Amazon/Netflix).

Salud

Análisis de genoma, predicción de epidemias, monitoreo de pacientes en tiempo real.

Finanzas

Detección de fraude, análisis de riesgo crediticio, trading algorítmico de alta frecuencia.

Deportes

Moneyball: Análisis de rendimiento de jugadores, prevención de lesiones mediante sensores.

1.4 - 1.5

Modelos, Tareas y Técnicas

Para extraer valor, aplicamos diferentes enfoques matemáticos y computacionales.

Aprendizaje Supervisado
Enseñamos al modelo con datos etiquetados (ej. fotos de gatos vs perros).
  • Clasificación: ¿Es A o B?
  • Regresión: Predecir un valor numérico (precio de casa).
Clustering (No Supervisado)
El algoritmo encuentra patrones ocultos sin etiquetas previas.
  • K-Means: Agrupar clientes por comportamiento de compra.
Procesamiento de Lenguaje Natural (NLP)
Técnicas para entender texto humano.
  • Análisis de Sentimiento: ¿El tweet es positivo o negativo?
  • Chatbots: Como el que usas ahora.

1.9 - 1.10

Gobernanza y Retos

Gobernanza de Datos

Es la gestión de la disponibilidad, integridad, usabilidad y seguridad de los datos.

  • Calidad: Datos limpios y fiables.
  • Seguridad: ¿Quién accede a qué?
  • Privacidad: Cumplimiento legal (GDPR).

Problemática y Desafíos

No todo es perfecto. Implementar Big Data trae problemas técnicos y éticos.

  • Escasez de talento especializado (Data Scientists).
  • Costos elevados de infraestructura.
  • Silos de datos desconectados.
